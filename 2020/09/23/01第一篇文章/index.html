vue初始化

1. {{}}是vue渲染数据状态的标签
    2.  标签中可以直接渲染data中的数据，也可以写简单的逻辑
    3. vue初始化的对象叫做vue的根组件

    5. 只有根组件的data是一个对象 data用来存储vue的数据状态




## vue的指令
1. vue的指令是vue的一个核心功能他可以帮助开发者快速高效的完成很多的逻辑
2. vue指令的写法基本都是v-指令明的形式 指令都是用在标签的开始标签后面
3. vue的指令后面的值是变量 不需要使用 {{}} 标签包裹

v-html 会把变量中的标签解析出来 让浏览器直接渲染标签中的内容 它会把对应的标签结构解析在使用该指令的元素内部

v-text会把对应的值当成字符串直接渲染出来 不会解析标签

v-show的作用

  • str1为true时显示

  • str2为false时隐藏 该元素会添加一个display:none的样式

  • 也就是说v-show是通过css样式控制元素的显示和隐藏的

    v-if的作用

  • 为true时显示会新增对应的元素

  • 为false时把元素给删除掉

  • 也就是说v-if是通过新增和删除dom控制元素的显示和隐藏

    v-for循环指令(数据迭代的指令)v-for循环的所有参数都可以自定义命名

  • v-for循环数组时 第一个参数item是数组项，第二个参数index是数组的索引值

  • itme是一个变量可以随意命名 但是不能写中文 item代表是数组项

  • index也是一个变量可以随意命名

  • v-for 循环对象时 有三个参数 第一个参数是对象的值 第二参数是对象的属性 第三个参数是对象的索引值(下标)

    v-on指令

  1. 在vue中可以通过v-on指令去绑定一个事件包括鼠标事件和键盘事件以及自定义事件
  2. v-on绑定的函数后面的()可写可不写
  • 当不写()的时候 函数有一个默认参数该参数就是事件的事件对象
  • 当写() 参数需要传递如果还需要事件对象则可一个函数传递一个$event该参数就是事件的事件对象 该参数不能随意命名
  1. v-on 可以缩写成@
  2. 事件还有各种修饰符
  • stop:

    调用 event.stopPropagation() 阻止单击事件继续传播

  • prevent:

    调用 event.preventDefault() 提交事件不再重载页面

  • capture:

    添加事件侦听器时使用 capture 模式。 添加事件监听器时使用事件捕获模式 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理

  • self:

    只当事件是从侦听器绑定的元素本身触发时才触发回调 只当在 event.target 是当前元素自身时触发处理函数 即事件不是从内部元素触发的

  • {keyCode | keyAlias}

    只当事件是从特定键触发时才触发回调。

  • native

    监听组件根元素的原生事件。

  • once:

    只触发一次回调。

  • left

    (2.2.0) 只当点击鼠标左键时触发。

  • right

    (2.2.0) 只当点击鼠标右键时触发。

  • middle

    (2.2.0) 只当点击鼠标中键时触发。

  • passive:

    (2.3.0) 以 { passive: true } 模式添加侦听器

    vue事件有很多修饰符 修饰符通过在事件后面用，修饰符的形式来写 修饰符可以串联

  • 例：@click.stop.prevent.self

    v-bind:是一个绑定指令，可以绑定标签中的所有属性 也可以绑定自定义属性

    v-bind：也可以缩写 直接不写v-bind 例
